# `text-map` 与 `EraMapTool` 对照与演进记录

## 项目定位

- 当前项目的 `text-map` 是浏览器内的轻量文字地图编辑器，擅长快速拼装字符地图、边框、字符 UI 和带简单样式的文本图。
- `EraMapTool` 是面向 Emuera 地图生产的完整桌面工具，覆盖地图绘制、区域操作、导入导出、图片参考、批量处理和动态颜色语义。

## 功能对照表

| 对比项 | 当前项目 `text-map` | `EraMapTool` |
| --- | --- | --- |
| 产品形态 | 浏览器内工具页 | 独立 Godot 桌面工具 |
| 主要定位 | 轻量文字地图 / 字符画 / 富文本示意 | Emuera 地图生产工具 |
| 画布尺寸 | 默认 `10x10`，最大 `40x40` | 固定 `64x42` |
| 数据模型 | 每格保存字符与样式 | 分离字符矩阵与颜色矩阵 |
| 全角/半角处理 | 处理显示宽度 | 原生支持 1 字节 / 2 字节混排 |
| 编辑模式 | 文本编辑、矩形填充 | 绘制、颜色、线、框、填充、区域拖拽 |
| 文本样式 | 颜色、加粗、下划线、删除线 | 以字符和颜色为主 |
| 复制粘贴 | 单行 / 选区 / 整图复制，支持 HTML | 更偏文件导入导出 |
| 导入 | 纯文本 | JSON、ERB、旧格式、拖放文件、拖放图片 |
| 导出 | ERB | JSON、ERB、PNG、截图 |
| 撤销重做 | 有，约 120 步 | 有，约 24 步 |
| 区域块操作 | 只有矩形填充 | 完整选区抓取、预览、拖动粘贴 |
| 行列批处理 | 无专门能力 | 有整行、整列、插入、删除、推挤 |
| 图形绘制 | 只有矩形涂块 | 线、矩形边框、区域填充 |
| 颜色系统 | 普通前景色 | 普通色 + 语义化动态色 |
| 图片辅助 | 无 | 拖图生成参考或草稿 |
| 批量处理 | 无 | 有 |

## 值得补到当前 `text-map` 的能力

### 高优先级

- `JSON` 工程文件导入导出。
  这样可以无损保存字符与样式，避免当前只有纯文本导入、ERB 导出的单向工作流。
- 多行块选区的复制、移动、粘贴。
  这是地图编辑器和普通文本框之间最明显的分界能力。
- 行列级操作。
  包括整行清空、整列填充、插入列、删除列、推挤行列。
- 线段、矩形边框、连续填充。
  对边框、道路、房间轮廓、表格结构等场景会明显提效。

### 中优先级

- 最近使用字符 / 自定义笔刷历史。
- 图片拖入作为参考层。
- 更完整的快捷键体系。
- PNG 预览导出。

### 暂不优先

- 季节、噪声、概率驱动的动态渲染系统。
- 批量文件转换。

## 本次实际落地事项

- 新增 ERB 导入，并与现有 ERB 导出格式配套。
  目标是让当前工具导出的 ERB 能被当前工具再次导入，并恢复字符、颜色、加粗、下划线、删除线。
- 拆分 `js/text-map.js`。
  目标是把文案与常量、页面渲染、数据转换、编辑器运行时拆开，降低后续改动时的误伤概率。
